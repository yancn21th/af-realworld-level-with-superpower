Design Section 3: Backend & Frontend Structure

Hono routes · React pages · Error handling · Testing

Backend — Hono Route Layout

apps/api/src/routes/
Route files
auth.ts     → POST /users, POST /users/login
user.ts     → GET/PUT /user
profiles.ts → GET /profiles/:username
              POST/DELETE .../follow
articles.ts → GET/POST /articles
              GET /articles/feed
              GET/PUT/DELETE /articles/:slug
comments.ts → GET/POST/DELETE .../comments
favorites.ts→ POST/DELETE .../favorite
tags.ts     → GET /tags
Middleware
auth.middleware.ts
  — verifies JWT Bearer token
  — sets c.var.userId
  — optional (some routes public)

error.middleware.ts
  — catches thrown errors
  — returns { errors: { body: [...] } }
  — maps to 401/404/422 per spec

Error Handling & Testing Strategy

API Error Responses

All errors → { errors: { body: ["msg"] } }
401 — missing/invalid JWT
403 — forbidden (not owner)
404 — article/user not found
409 — email/username taken
422 — validation failed

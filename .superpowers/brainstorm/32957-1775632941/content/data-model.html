Key design decisions

  • Tags are stored inline in article_tags (no separate tags table) — keeps queries simple, matches spec's flat tag list
  • Timestamps stored as INTEGER (Unix ms) — SQLite-native, no timezone issues
  • Slugs are slugify(title) + '-' + randomId to ensure uniqueness on duplicate titles
  • Passwords hashed with bcrypt (cost 10)
  • JWT tokens: signed with HS256, payload = { sub: userId }
